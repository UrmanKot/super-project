QC-WAREHOUSE

This section includes invoices and orders from own production that have passed QC.
They are categorized

Open the necessary invoice or order for your production, select items, select the locator where you want to place the selected item. Locators that already have positions of this item are marked in blue. Within the same warehouse, we can only select a locator, where position data is already located.

PRODUCTION LIST

A production list is an assembly sent to production, for which it is necessary to check the availability of the necessary items in the warehouse, send a request for the purchase of purchased items and prepare all available items on one locator, so that later, when production requests the necessary items for sub assemblies, quickly prepare them and put into production

It is necessary to put down the actual quantity when creating a new production list, as well as track the receipt of items that were previously in deficit, and their transfer to the production list locator. Items that were previously in deficit, but then received at the warehouse and became available, are marked in color
red - available items that are not related to this production list (manually ordered to the warehouse) green - items received at the warehouse, ordered directly through this production list or through the plan (auto request)
If, when preparing a production list, we decide to use a ready-made assembly included in this production list, there is no need to prepare the items included in this assembly.

PRODUCTION REQUEST

Production requests - requests from production for certain items needed to produce specific sub- assemblies of the production list or to produce specific manufactured items at the moment

It is a list of required items, subassemblies or material, indicating the required quantity and location in the warehouse

It is necessary to prepare, print out the form, sign the form yourself and at the production site, keep a copy for yourself, complete the request, transfer the positions according to the list to production. It is possible to cancel a production request if it was created by mistake. it is possible to use scanning when searching for items in the warehouse

TOOL REQUEST

A list of tools or supplies requested by the employee for use or requested for the department. During processing, they are moved to the employee's (department's) locator

WHERE USED

Filter to find which builds use the selected stock item

PHYSICAL INVENTORY

Allows you to make an inventory of the selected locators in order to bring the number in the erp system in line with the real number

To create a new physical inventory, select the warehouses and shelves for which you need an inventory

After creation, a list of all positions recorded on the selected locators will be displayed. It is necessary to bring the recorded quantity into conformity with the real one. To do this, you need to edit the entered quantity field. If during the inventory process it is necessary to inventory an item that is not in the list, you need to click the add product button, find the required item, enter the quantity, select the locator to which we want to record it, and enter the technology if it is a manufactured item or assembly.

Also, if you need to change the position locator in the inventory list, you can do this by clicking the move product button. After the completion of the inventory, you must click the Complite button, after which all changes will be written to the system

Warehouses

It is possible to create the required number of warehouses - storage areas with their own structure locators.

Each warehouse has locators — specific storage locations within the warehouse. Each locator, in addition to the name and description, can have coordinates - a row, a column, a shelf, a place.

Unit of measure

Units of measure to be used for warehouse categories

Serial numbers type

You can assign a serial number to a warehouse item. The assignment of a serial number occurs when completing an invoice (purchase chains and chain outsourcing), ordering to your own production or when confirming production (a serial number is assigned only to the root assembly)

In order for the item to have a serial number, you need to create a new type of serial number, in which you need to fill in the prefix - the invariable part of the serial number, code - the part of the number that will increase by one when creating a new one, and the suffix - the invariable part of the serial number.

And then select an item that will obey the created serial number assignment rule.
